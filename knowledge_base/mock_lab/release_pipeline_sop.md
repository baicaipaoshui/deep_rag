# 发布流程 SOP（Aurora 项目）

## 适用范围
适用于 Aurora 服务（API 网关、分诊服务、复盘服务）在生产环境的常规发布与紧急修复发布。

## 标准流程
1. 需求冻结：确认版本范围、变更清单、回滚方案。
2. 质量门禁：单元测试 >= 85%，关键链路回归通过率 100%。
3. 灰度准备：配置开关、流量分组、监控看板与告警阈值。
4. 阶段发布：按 5% -> 30% -> 100% 放量，每阶段观察 2 小时。
5. 异常回滚：触发条件包括错误率 > 2%、p95 延迟 > 1.2s、超时率 > 5%。
6. 发布复盘：记录问题、根因、改进项，沉淀到复盘知识库。

## 角色职责
- 发布经理：统筹排期、主持发布、决策回滚。
- 值班 SRE：实时监控核心指标，执行回滚操作。
- 研发负责人：问题定位、修复确认、补丁交付。

## 核心指标阈值
- 成功率 >= 99.5%
- p95 延迟 <= 800ms
- 工单自动闭环率 >= 90%

## 例外流程
紧急漏洞修复可跳过 30% 灰度阶段，但必须满足：
1. 双人审批；
2. 回滚预案已演练；
3. 风险公告已同步到值班群。
