# 前端性能优化指南

## 一、加载性能优化

### 1.1 代码分割与懒加载

当前项目的主包体积已经达到2.8MB（gzip后680KB），严重影响首屏加载速度。建议采取以下措施：

- 使用React.lazy和Suspense对路由级组件进行懒加载，预计可将首屏包体积减少60%
- 对大型第三方库（如echarts、moment.js）使用动态import按需加载
- 将不常用的管理后台模块独立成单独的chunk

实测数据：在测试环境中，代码分割后首屏加载时间从4.2秒降至1.8秒。

### 1.2 资源压缩与优化

- 启用Brotli压缩替代gzip，压缩率可提升15-20%
- 图片资源使用WebP格式，平均体积可减少30%
- 使用CSS Modules替代全局CSS，减少未使用样式的打包

### 1.3 CDN与缓存策略

- 静态资源部署到CDN，配合内容哈希实现长期缓存
- 设置合理的Cache-Control：HTML文件设为no-cache，JS/CSS文件设为max-age=31536000
- 使用Service Worker实现离线缓存，提升二次访问速度

## 二、渲染性能优化

### 2.1 减少不必要的重渲染

当前存在的问题：商品列表页在滚动时会触发整个页面的重渲染，帧率降至20fps以下。

建议措施：
- 使用React.memo包裹列表项组件，避免父组件state变更导致所有子组件重渲染
- 将频繁变化的状态（如滚动位置、鼠标坐标）使用useRef替代useState
- 对复杂计算使用useMemo缓存结果

### 2.2 长列表虚拟化

商品列表页目前一次性渲染全部500+条数据，DOM节点数量过多导致卡顿。

建议使用react-window或react-virtualized实现虚拟列表，只渲染可视区域内的元素。实测在500条数据的场景下，DOM节点从500+降至15-20个，滚动帧率从18fps提升至58fps。

### 2.3 动画性能

- 避免使用触发布局的CSS属性（如width、height、top、left）做动画
- 优先使用transform和opacity，它们可以在GPU上加速
- 使用requestAnimationFrame替代setTimeout实现JavaScript动画

## 三、网络请求优化

### 3.1 请求合并与预加载

当前商品详情页需要发送6个API请求才能完成渲染，总耗时约3秒。

建议：
- 将商品基本信息、价格、库存三个接口合并为一个BFF接口，减少请求数
- 使用预加载（link rel="preload"）提前加载关键API响应
- 在列表页hover时预请求详情数据

### 3.2 数据缓存

- 使用SWR或React Query实现请求级缓存，避免重复请求
- 设置合理的staleTime，对于不频繁变化的数据（如商品分类）设为5分钟
- 对分页数据实现前后页预加载

### 3.3 接口响应优化

- 使用GraphQL或字段选择机制，避免返回前端不需要的字段
- 对大型响应（如商品列表）实现分页加载，每页20条
- 关键接口启用HTTP/2 Server Push

## 四、监控与度量

### 4.1 性能指标

建议关注以下核心Web Vitals指标：
- LCP（最大内容绘制）：目标 < 2.5秒
- FID（首次输入延迟）：目标 < 100ms
- CLS（累计布局偏移）：目标 < 0.1

### 4.2 性能监控方案

- 使用Performance Observer API采集实际用户性能数据
- 每周生成性能报告，跟踪趋势变化
- 设置性能预算：主包体积不超过500KB（gzip后），首屏LCP不超过3秒
